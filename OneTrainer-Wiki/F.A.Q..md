* 我可以把训练照片数据集放在哪里？
> 在概念选项卡中。首先添加一个配置，然后向此配置添加一个概念。当您单击概念时，它会打开一个窗口，您可以在其中指定数据集的路径。您可以为一个配置定义多个概念。

* 支持哪些图像类型？
> 支持的完整扩展名列表：
['.bmp', '.jpg', '.jpeg', '.png', '.tif', '.tiff', '.webp']

* 我可以把触发词放在哪里？
> 嵌入提示中使用的触发词就是文件名，LoRA 没有触发词。
> 对于嵌入和额外嵌入，描述必须包含占位符，否则图像不会被训练。

* [嵌入] 训练嵌入和额外嵌入时，我应该如何使用占位符 `<embedding>`？
> 它只是一个占位符，您可以使用任何单词，在额外嵌入选项之前，它必须严格是 `<embedding>`，但现在您可以使用任何您想要的单词，只需要与您在描述中设置的触发词匹配。

* [嵌入] 如何设置初始嵌入文本？
> 简而言之，初始嵌入文本应该描述您的主题，但不应在 token 数量上超过嵌入 token 数量。

> 这里有一个解释。让我们以您的示例提示 "photograph of `<embedding>` with" 为例，并且（为简单起见）假设每个单词都编码为单个 token。这可能会产生以下 token ID：[1, 2, ..., 3]，其中 "..." 是嵌入插入的位置。如果您有一个 3 token 的嵌入，例如它会是 [1, 2, 100, 101, 102, 3]。

> 现在假设您设置了一个初始文本 "blond woman wearing a pink hat"。那是 6 个 token。但嵌入只支持 3 个 token，所以实际上只使用了前 3 个（6 个中的）。其余的被截断。

> 这也反过来。如果您只提供一个较短的文本（如 "blond woman"），它不知道用什么作为第三个 token。在 OneTrainer 中，token 用 " * " token 填充，所以 "blond woman" 变成 "blond woman *"。

* 嵌入在质量/灵活性方面是否与 LoRA 相当？
> 嵌入（Textual Inversion）反向工作以找到与您的图像匹配的 token，因此它只能学习基础模型已经知道的东西。因此它们可以非常灵活。它们对人物很好，因为模型已经见过很多不同的人。但它不能学习全新的东西。
它们不需要很多图像，15-30 张图像就足够了，即使是极简数据集（约 6 张图像）_也可以_工作。
另请注意，它们可以与 LoRA 一起使用。

> LoRA 为模型添加了一组权重。它们通常用于人物、风格、姿势和图像构图。根据您训练的内容，您可以通过添加具有不同姿势/构图的图像来使它们更灵活...它们可以用 20 张图像的小数据集进行训练，但为了灵活性，您需要更多。

* 训练需要多少 VRAM？
> 这取决于您在训练什么以及参数。SD1.5 LoRA 或 512 分辨率的嵌入可以只用 6Gb 进行训练。但对于大多数情况，您需要 10-12 Gb。对于 SDXL 微调，24Gb 真的不足以在不妥协的情况下完成。

* 我非常喜欢像在 Kohya 中那样为我训练的 lora 使用类别图片/正则化图像。
> 类别图像是一个有点虚构的概念。它们只是没有复杂描述的常规图像，通常以较低的权重训练。您可以在 OneTrainer 中轻松做到这一点，将它们添加为新概念，设置较低的重复次数，也许稍微降低损失权重。对于描述，选择"来自单个文件"，并将您的类别提示放在该文件中
