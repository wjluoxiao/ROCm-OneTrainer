# 前言

本指南不涵盖

# 模型

根据我的经验，大多数人将精力集中在 SD 1.5 和 SDXL 上。SD2.x 被广泛认为很糟糕，不惜一切代价避免它。Stable Cascade 被有研究兴趣的人关注，因为它在微调方面有某些优势。

衍生模型应该能够使用基础模型的设置进行训练。例如，SDXL Lightning 和 Pony 应该能够使用 SDXL 配置进行训练。这并不是说用于 Pony 的 LoRA 在从基础模型训练时会工作，而是说基础模型为 Pony 的 SDXL 设置应该是一个可行的解决方案。

## 注意事项
* Stable Cascade 目前作为纯研究模型存在固有的挑战。这意味着分发和生成很困难。这可能将其限制在自定义 ComfyUI 工作流中。注意，SC 的标准 StableSwarm 包与原始分发不同。
* Pixart Alpha 在发布时寄予厚望，但 T5 的使用使其成为消费级 GPU 上只能训练 unet 的模型
* SD2.x 使用 v-prediction 训练，曾有希望但未能实现。几乎所有修复它的尝试现在都已放弃

# 训练类型
目前 OneTrainer 中有 4 种，很快将有 5 种训练类型

* 文本反转（简称为嵌入）。加载整个模型（VRAM 要求），但计算速度很快，因为程序试图找到构成您训练概念的 token。嵌入是小文件，因为它们只是 token。
* LoRA（大语言模型的低秩适应）。可以加载部分或整个模型（可变 VRAM 要求），创建将位于基础扩散模型之上的层。LoRA 可以训练 unet、文本编码器或两者。由于 LoRA 位于扩散模型之上，几乎不可能不让 LoRA 渗透到基础模型的其他非预期部分。
* 微调。可以加载部分或整个模型（可变 VRAM 要求）并直接受训练影响。为了防止破坏基础模型，尽可能最小化梯度更新非常重要。这需要大的批量或虚拟批量大小，因此微调需要最好的硬件才能在不延长到数周或数月的时间内进行有意义的微调。
* 修复训练。与微调相同，但用于修复工作流。我没有这方面的经验，所以无法分享太多细节。

## 训练总结
* 嵌入 - 找到解决您概念的 token。文件很小，计算速度快，但根据您的概念可能很难找到正确的解决方案。
* LoRA - 在现有模型之上构建新模型。通常是分发的最佳选择。大小可变，取决于网络秩。LoRA 可以合并到检查点中。
* 微调 - 直接调整或扩展现有模型。通常是一个非常缓慢的过程，以免破坏现有模型。为多功能和强大的微调构建数据集可能需要一个人很长时间，但一旦构建完成，随着技术的变化，该数据集可用于训练未来的模型。大玩家可以用数量和商业硬件做的事情，可以通过更好、高质量的专注工作来完成。

# OneTrainer 通用工作流
按以下顺序设置您的训练运行可能会有帮助。

## 准备您的环境
1. 确定您想要的模型和训练并加载模板
2. 确定要使用的模型（来自 huggingface 的基础模型还是不同的检查点？）
3. 设置您的工作环境（文件夹）

## 准备您的训练
1. 设置您的概念
2. 设置您的样本
3. 如果需要，设置您的保存和备份频率
4. 设置您的最终模型保存位置

## 决定您的模型和训练细节
1. 根据您的 VRAM 预算和硬件支持确定您的模型权重。
2. 根据您的 VRAM 预算和硬件支持确定您的训练数据类型。
3. 确定您的训练分辨率，理想情况下基于您正在使用的模型。（例如，SDXL 为 1024，SD1.5 为 512）
4. 根据概念数量、VRAM 预算和硬件支持确定您的批量大小
* 您的批量应该能被一个 epoch 中使用的图像整除。这不是一个硬性要求，但同时如果不能整除，您不知道哪些图像会被丢弃。
* 您应该在一个宽高比分桶中有足够的图像来填满一个批次。如果没有，该分桶将永远不会被训练。OneTrainer 目前不显示此信息，您得到的唯一反馈是步骤 x 批量是否等于您的概念图像。


### 注意事项
* GTX 1000 和更旧的显卡不支持 BF16 权重或计算。
* RTX 3000 和更新的显卡需要进行 TF32 计算。注意：检查 TF32 与 FP32 的性能。
* RTX 4000 和（假设）更新的显卡支持 FP8。较旧的显卡可能受益于内存节省，但不会有速度提升。

### FP32 模型权重 - 大约
* Stable Diffusion v1.5 (512px) - Unet: 3.44GB, TENC: .48GB, VAE: .32GB, 总计: 4.27GB
* Stable Diffusion v2 (512px) - Unet: 3.46GB, TENC: 1.36GB, VAE: .32GB, 总计: 5.21GB
* Stable Diffusion v2 (768px) - Unet: 3.46GB, TENC: 1.36GB, VAE: .32GB, 总计: 5.24GB
* Stable Diffusion XL (1024px) - Unet: 10.3GB, TENC1: .48GB, TENC2: 2.78GB, VAE: .32GB, 总计: 13.88GB。注意：SDXL 几乎总是以 6.94GB fp16 模型权重分发。即使是 Huggingface 也只有 fp16 组合的 safetensors。
* Stable Cascade (1024px, Stage C - 36 亿参数) - Stage C: 14.4GB, TENC: 1.39GB, 总计: 15.69GB
* Stable Cascade (1024px, Stage C - 10 亿参数，标记为 'lite') - Stage C: 4.2GB, TENC: 1.39GB, 总计: 5.59GB
* 注意，Stable Cascade TENC 似乎已经是 BF16 权重。
* fp16 或 bf16 将把 VRAM 中需要的权重减少一半。fp8 将把文件大小减少到原来的四分之一。

## 决定您的优化器和训练细节
1. 根据您的训练类型和 VRAM 预算选择您的优化器
2. 根据您的优化器和训练类型选择您的学习调度器
3. LoRA - 转到 LoRA 选项卡并确定您的网络秩、alpha 和 dropout
4. 嵌入 - 转到嵌入选项卡并确定您的 token
5. 微调 - 不需要特殊的选项卡或设置。

## 检查您的优化器设置
1. 单击优化器旁边的 ... 并检查启用或输入的设置，请记住每个优化器最后使用的设置将默认出现。
2. 查看 [这里](https://github.com/Nerogar/OneTrainer/wiki/Optimizers) 了解这些设置的更多信息。

## 决定您的学习率、Epochs、预热
1. 根据您的训练类型和优化器输入您的主要学习率
2. 根据您的优化器、训练类型和调度器选择 epoch 的数量
3. 根据您的优化器和训练类型决定预热步骤的数量
4. 决定是否要训练文本编码器，训练多长时间，以及学习率
5. 决定是否要训练 Unet（或 SC 的 prior），训练多长时间，以及它的学习率。
