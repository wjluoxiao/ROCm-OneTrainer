# 概念选项卡概览

概念（Concepts）选项卡用于定义 OneTrainer 的数据集。概念是任意的数据桶 + 定义增强方式，它们可以包含训练数据、'reg'（正则化）数据或验证数据。拥有多个概念是很常见的情况。

## 概念选项卡界面

![image](https://github.com/Nerogar/OneTrainer/assets/132208482/2d02d53d-3937-444e-9b6a-9c8930924f57)

该选项卡包含以下元素：

- **下拉菜单（默认：concepts）：**  
  在多个概念配置之间进行选择。OneTrainer 仅使用当前选中的配置进行训练。

- **添加配置（Add Config）：**  
  打开一个界面，输入名称并创建新的概念配置。

- **添加概念（Add Concept）：**  
  在当前配置中创建一个新的、默认的空白概念。

- **删除概念（Delete Concept，红色 X）：**  
  从配置中移除一个概念。

- **复制概念（Duplicate Concept，绿色加号）：**  
  复制现有概念及其所有设置。

- **启用概念（Enable Concept，开关，默认：开启）：**  
  切换是否使用此概念进行训练。启用时，开关显示为蓝色。

- **编辑概念（Edit Concept）：**  
  点击概念图片的任意部分（按钮和开关除外）会打开设置窗口以进行进一步修改。


## 概念设置 - 通用

![image](https://github.com/user-attachments/assets/9071e1eb-c300-4090-baf3-565a3da7a698)

本节涵盖基本信息、平衡和缓存设置。

- **名称（Name，默认：空白）：**  
  为您的概念输入名称。如果留空，关闭窗口时将默认为提供的文件夹名称。

- **已启用（Enabled，默认：True）：**  
  与上一个选项卡（概念）上的开关相对应；它控制 OneTrainer 是否使用此概念进行训练。

- **概念类型（Concept Type，默认：Standard）：**  
  下拉菜单，您可以选择此概念是常规训练概念、验证概念（用于验证损失）还是先验预测概念（用于先验预测模型保留）。关于验证，请参阅 [验证如何工作](https://github.com/Nerogar/OneTrainer/wiki/How-Validation-works) 了解更多信息。关于先验预测的更多信息，请参阅 [先验预测](https://github.com/Nerogar/OneTrainer/wiki/Prior-Prediction)。

- **路径（Path，默认：空白）：**  
  输入或粘贴概念图片的路径。您也可以点击（...）按钮浏览文件夹。

- **提示词来源（Prompt Source，默认：from text file per sample）：**  
  从三个选项中选择一个：
  1. **每个样本来自文本文件（From text file per sample）：**  
     例如，`0001.jpg` 使用 `0001.txt` 文件作为其提示词。多个描述（每行一个）将允许 OneTrainer 在每个 epoch 中随机选择一个。
  2. **来自单个文本文件（From single text file）：**  
     使用一个文本文件作为所有图像的提示词。
  3. **来自图像文件名（From image file name）：**  
     例如，`tag1 tag2 tag3.jpg` 使用 "tag1 tag2 tag3" 作为提示词。

- **包含子目录（Include Subdirectories，默认：False）：**  
  将子目录中的图像视为单个概念的一部分，以便于管理。

- **图像变体（Image Variations，默认：1）：**  
  控制使用增强变量缓存多少张图像。使用图像增强和潜在缓存时需要此设置。

- **文本变体（Text Variations，默认：1）：**  
  定义为每张图像缓存多少个提示词变体。使用文本增强或多行描述时需要此设置。（注意：训练文本编码器（TE）或嵌入时，提示词不会被缓存，因此此设置不是必需的。但是，如果您进行了缓存但不训练 TE 或嵌入，则必须增加变体数量，否则将始终相同）

- **平衡（Balancing，默认：1 Repeats）：**  
  平衡每个概念的贡献。例如，当您有 10,000 张正则化图像和 100 张源图像时，这很有用。有两种模式：
  - **重复次数（Repeats，默认：1 即无）：**  
    每个 epoch 将源图像的数量乘以定义的值。
  - **采样（Sample）：**  
    每个 epoch 使用精确数量的随机选择图像，无论可用总数是多少。
  - **关于图像选择方法的说明：**  
    图像选择是确定性的，但带有随机性。例如：对于 100 张图像的概念，repeat 0.2 或 20 个样本将在第一个 epoch 随机选择 20 张图像，然后从剩余的 80 张中再随机选择 20 张，依此类推。这意味着所有图像将在 5 个 epoch 后全部被使用。


- **损失权重（Loss Weight，默认：1）：**  
  调整每个概念在训练过程中的影响。较低的值可用于正则化图像等概念，如果它们的影响过大的话。

## 图像增强选项卡

![image](https://github.com/user-attachments/assets/f2b023ff-d77a-4b2c-ade3-a1349999757a)

此选项卡提供通过图像增强来丰富数据集的选项，这对于小数据集尤为重要。增强可以是随机的或固定的——此设置需要要么禁用潜在缓存（通过每个 epoch 缓存），要么使用图像变体。

- **更新预览（Update Preview）：**  
  生成图像和文本增强的预览。预览图像下方的开关可以禁用预览的增强，如果您需要查看未修改的图像和描述。对于随机设置，重复点击将让您更好地了解增强的应用强度。

- **裁剪抖动（Crop Jitter，默认：开启）：**  
  如果 OneTrainer 需要裁剪图像，此选项会应用随机的非居中裁剪以增加多样性。

- **随机翻转（Random Flip，默认：开启）：**  
  沿垂直轴翻转图像，可以是随机的或固定的。

- **随机旋转（Random Rotation，默认：关闭 - 0）：**  
  将图像随机旋转最多指定的度数，或者如果设置了则使用固定旋转。

- **随机亮度（Random Brightness，默认：关闭 - 0）、随机对比度（Random Contrast，默认：关闭 - 0）、随机饱和度（Random Saturation，默认：关闭 - 0）、随机色相（Random Hue，默认：关闭 - 0）：**  
  每个设置都会通过随机方差或固定值调整相应的图像属性。
  
- **分辨率覆盖（Resolution Override，默认：关闭 - 512）- 此功能可用于覆盖概念的训练分辨率。禁用时，One Trainer 将图像重新缩放到您的训练分辨率。换句话说，当分辨率覆盖关闭时，图像会被放大/缩小到训练分辨率；当分辨率覆盖开启时，图像会被放大/缩小到分辨率覆盖字段中设置的值。


 可能的值：可以是单个分辨率或用逗号分隔的多个分辨率，或者是宽 x 高格式的单个分辨率。如果您设置了多个分辨率，分辨率选择是随机的，您需要增加图像变体。但请记住，缓存图像时分辨率选择仍然是随机的，这意味着有 2 个分辨率且图像变体设置为 2 时，相同的分辨率可能会被缓存两次。另请注意，概念中的每张图像都会随机应用其中一个分辨率。
  - 激活后，它可用于两个目的：
    - 使用 [多分辨率训练](https://github.com/Nerogar/OneTrainer/wiki/Lessons-Learnt-and-Tutorials#multi-resolution-training)（用逗号分隔的多个训练分辨率）时，它将使用相同分辨率的概念图像。
    - 为了防止图像放大，您可以以 1024（目标分辨率）训练，而 512 或 256 的图像不会被放大。训练将在 512 或 256 下进行。这对低质量图像有帮助。

# 文本增强选项卡

![Screenshot 2024-11-09 215015](https://github.com/user-attachments/assets/54955edf-89f2-4087-84a5-c25c59d70db3)

文本增强会修改每张图像的描述。描述是由第一个选项卡上的"提示词来源"设置定义的完整文本，并使用自定义的**分隔符**（通常是逗号）拆分为"标签"。每个 epoch 都会为每张图像生成新的变体，因此使用更多 epoch 且每个概念的重复/样本更少可以最大化文本变化。这些变体有助于防止过拟合并提高提示词的灵活性。根据模型的不同，您可能会通过基于标签或自然语言的描述获得更好的结果。

### 标签打乱

随机化描述中标签的顺序。这有助于抵消放置在开头的标签的任何意外重要性。**保留标签数量（Keep Tag Count）** 设置确保指定数量的标签保留在前面，保留关键的"触发词"（注意这些在 OT 中不是真实存在的东西）。

**保留标签数量** 将指定始终保留在描述前面的标签数量。如果在特定概念上训练 LoRA，最好将该概念的名称（即"触发词"）保留在前面，以使训练更专注于它。

### 标签/描述丢弃

随机移除一些标签，以帮助提高较短提示词的性能。但是，丢弃太多标签可能会降低模型区分概念的能力。**保留标签数量** 和**分隔符**设置都适用于此过程。

**概率（Probability）** 设置（取值在 0 到 1 之间）控制丢弃的几率，有三种模式可用：
- **全部（Full）：**  
  要么以设定的概率移除所有标签（除了由**保留标签数量**保留的标签），要么保持描述不变。
- **随机（Random）：**  
  单独评估每个标签是否可能被移除。
- **随机加权（Random Weighted）：**  
  功能类似于**随机**，但降低了丢弃开头标签的几率（线性增加到末尾）。

您还可以定义**特殊标签（Special Tags）** 列表。这些标签是：
- **白名单：** 始终保留在描述中，其他所有内容都可能被丢弃
- **黑名单：** 只有这些标签会被丢弃，而其他标签保持不变。  

特殊标签可以以逗号分隔列表的形式提供，或者作为文件路径（.txt 或 .csv 格式），每个新条目占一行。如果启用了**特殊标签正则表达式（Special Tags Regex）**，您可以使用正则表达式模式（例如 "photo.*"、"\\d.*"、"d.{1}g"）来匹配相关标签。

*注意：* 如果标签包含正则表达式特殊字符（如 . ^ $ * + ? ! { } [ ] | ( ) \\），请用反斜杠转义它们（例如 "Panic\\! at the Disco"）。

### 随机大小写

此功能对描述中的标签应用各种大小写样式。每个标签都有机会根据定义的概率获得以下样式之一：
- **全大写（capslock）：** 所有字母大写。
- **标题（title）：** 每个单词的首字母大写。
- **首字母（first）：** 只有第一个单词大写。
- **随机（random）：** 每个字母随机大小写。  
如果启用了**强制小写（Force Lowercase）**，则在应用任何其他更改之前，整个描述会转换为小写。请注意，使用 CLIP 的模型（如 SD1/2/XL）会忽略大小写变化，因为它们不区分大小写，而基于 T5 的模型（如 SD3、Flux）会受到影响。

## 概念统计

![image](https://github.com/user-attachments/assets/329b2a7d-9540-4c45-8130-3f66689a6916)


此选项卡显示有关概念文件和文件夹的详细统计信息。OneTrainer 在您选择选项后扫描概念文件夹，并将详细信息保存到配置文件中。如果内容发生变化，请手动运行扫描以更新信息。

- **基本统计（Basic Statistics）：**  
  快速计算图像（和视频）、描述和掩码文件的数量。还会报告总文件大小。此扫描速度很快，对平衡很有用。

- **高级统计（Advanced Statistics）：**  
  提供详细的见解，例如：
  - 图像、掩码和描述是否正确配对。
  - 最小、最大和平均图像分辨率（以百万像素和尺寸计）。
  - 描述的长度和字数。
  - 图像宽高比的分布。  
  此扫描更耗时，尤其是在硬盘存储上。将图像组织到子文件夹中（并启用"包含子目录"）可以提高速度。

附加细节：
- "配对的"掩码或描述与其对应的图像共享相同的基本名称（例如，"image1.jpg" 有 "image1-masklabel.png" 和 "image1.txt"）。未配对的文件表示存在问题，但不会影响训练，因为它们会被忽略。

- **最小/最大/平均像素（Min/Max/Average Pixels）：**  
  提供精确的尺寸。明显小于模型基础分辨率（SD1 为 512，SDXL 为 1024）的图像在放大时可能会导致伪影。

- **长度（帧）和 FPS（每秒帧数）：**  
  这些适用于视频。如果视频的 fps 与目标不匹配（例如 Hunyuan 为 24），可能会显得太快或太慢。

- **最小/最大/平均描述（Min/Max/Average Captions）：**  
  基于字符计数，并提供近似的字数。由于大多数模型基于 token 计数（每个单词大约 1-2 个 token），超过模型 token 限制的描述（例如基于 CLIP 的模型为 75 个 token）将被截断。

- **宽高比分桶（Aspect Bucketing）：**  
  显示图像宽高比的分布。OneTrainer 在训练期间将图像裁剪为一组宽高比分桶。批次由具有相同比例的图像组成，因此如果没有足够的特定宽高比图像来填满批次，那么所有这些图像都将被丢弃。为了避免这种情况，请手动调整图像或增加每个 epoch 的重复/样本数。

- **注意：**  
  如果在 Windows 的命令提示符中启动，统计选项卡功能可能无法工作，除非您在 Windows 10/11 上启用了 UTF-8 支持。
