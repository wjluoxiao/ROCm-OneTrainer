### 常用设置

* `工作区目录`（默认 - workspace/run）：必填，此目录将包含训练期间生成的样本、训练设置、备份和 tensorboard 日志。您可以使用单个文件夹或为每个项目创建一个文件夹。如果您处理多个项目/尝试，第二个选项可能会很有用。
* `缓存目录`（默认 - workspace-cache/run）：必填，在运行时用于存储缓存的图像和文本。
* `从上次备份继续`（默认 - 关闭）：启用时，将使用所选工作区中的上次备份。
* `调试模式`（默认 - 关闭）：切换到调试模式。这将为您提供 OneTrainer 正在执行的操作的数据和图像，包括训练步骤的图像（预测与实际）。注意，您可以在训练期间手动启用和禁用调试模式。
    * 调试图像是从 .pt（pickled tensors）文件生成的。这意味着它们在保存为图像时会经过模型的 VAE。
    * 因此，这些图像的质量可能低于预期，但它们不是经过 VAE 之前的图像。
* `调试目录`（默认 - debug）：可选，仅在开启调试模式时需要。
* `Tensorboard`（默认 - 开启）：训练开始时是否启动 Tensorboard。Tensorboard 用于查看训练运行的统计信息。当前工作区中的所有 Tensorboard 运行都可用于查看。
* `暴露 Tensorboard`（默认 - 关闭）：这会将 Tensorboard 从本地主机移动到暴露给网络的其他部分。
* `始终开启的 Tensorboard`：启动 Tensorboard 并允许您为已完成的训练打开它（来自当前工作区目录的 Tensorboard 日志）。
* `数据加载器线程`（默认 2）：在缓存期间使用。如果您的 GPU 可以处理，请增加它，这会增加缓存时间。
* `训练设备`（默认 - cuda）：用于选择要训练的 GPU 的自由文本字段，保留默认值或定义您要使用的 GPU，例如：cuda:1
* `临时设备`（默认 - cpu）：用于选择模型在不使用时驻留位置的自由文本字段。默认是 CPU 内存（RAM）。要禁用此选项，可以使用训练设备（即 cuda）。

### 验证

* 可选，请查看 [验证数据集](https://github.com/Nerogar/OneTrainer/wiki/How-to-setup-and-evaluate-validation-datasets) 以充分利用此功能。
* `验证`（默认 - 关闭）：这将启用验证步骤，如果您使用 Tensorboard，还会在其中添加一个卡片
* `验证间隔`：一个自由文本字段，用于输入您希望验证的频率。单位使用随附的下拉菜单进行选择。

----------
### 多 GPU

* 可以在多个 GPU 上进行训练，为此启用 `多 GPU` 并在 `设备索引` 中定义您要使用的 GPU，用逗号分隔，如果为空，它将使用所有可用的 GPU。`设备索引` 中的顺序很重要，它将使用第一个 GPU 作为主 GPU，例如 0,1 或 1,0。
* `顺序模型设置`：启用时，它将在每个 GPU 中顺序加载和设置基础模型，速度较慢但节省 RAM。
* `梯度减少精度`、`融合梯度减少`、`异步梯度减少`、`缓冲区大小（MB）`：默认值用于最大精度，但会使用最大的 VRAM 和每次操作的秒数。如果您想减少 VRAM 使用并加速操作，请阅读它们的工具提示并尝试调整这些参数。
* 关于批量大小的注意事项：训练选项卡中的字段已重命名为本地批量大小。使用多个 GPU 时，本地批次在每个 GPU 中执行，然后累积。因此，如果您计划在 2 个 GPU 上以 BS8 训练，请将本地批次设置为 4。
* Windows 用户注意事项：Windows 在处理多个 GPU 方面不如 Linux 高效。对于 Lora 和嵌入，这不是很明显，但对于微调会受到很大影响。强烈建议在 Windows 上进行多 GPU 微调时使用梯度减少设置。
