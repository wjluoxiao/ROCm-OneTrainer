这些是人们从 Kohya 过来时常犯的一些常见错误。

## 图像太少 / 批量大小太大

这通常会影响具有非常小（< 100 张图像）数据集且分辨率差异很大的人。Onetrainer 需要图像分辨率分桶来填充完整批次。如果该分桶没有填充批次，该分桶将被''丢弃''。Kohya 有不同的行为，如果分桶不足，它会复制其中的一些数据来填充批次。两种策略各有利弊，Onetrainer 和 Kohya 选择了不同的策略。

当您每个 epoch 的步骤数少于预期时，您可以判断是否有这个问题。在极端情况下，您甚至会看到每个 epoch 有 1 或 0 步。如果这个问题影响到您，您可以通过增加重复次数（复制数据以潜在地填充批次）或减少批量大小来解决它。

## 精度错误

如果您在 Kohya 中有一些工作但在 Onetrainer 中没有，请绝对确保您在两者中以相同的精度级别进行训练。BF16 和 F32 之间可能有''截然不同''的结果。三重检查您的配置。确保您在 Kohya 中设置的所有精度级别设置，您都在 Onetrainer 中设置了等效的。

## 缩放参数

如果您从头开始训练 LoRA 并使用 Adafactor，请关闭缩放参数。它有一个非常特定的用例，从头开始的 LoRA 训练不是这些用例之一。如果您保持开启，您可以训练一百个 epoch 但一无所获。

## 我如何计算步骤数，重复次数呢？！

重复次数仅用于平衡不平衡的概念，因此它们在 UI 中不称为重复次数，而是称为"平衡"，因此应保持为 1。鉴于此，"步骤公式"保持不变，但这最终是无用的，因为步骤数不等于好结果，您需要评估模型在 tensorboard 中输出的生成样本。

`((概念 1 的图像数 * 重复) + (概念 2 的图像数 * 重复) ... * epoch) % 批量大小`

## 帮助！我仍然有问题！

如果在检查了所有这些问题之后，您仍然无法在 Onetrainer 中获得与 Kohya 中结果匹配的结果，请执行以下操作：

* 从 Kohya 和 Onetrainer 导出您的 .json 配置。
* 逐行比较它们。确保您在 Kohya 中设置的所有设置都在 Onetrainer 中设置了。
* 如果在上述之后您仍然有麻烦，欢迎您来到 Discord 并在 #help 频道中提问。但请确保您先完成了上述两个步骤。配置将是我们首先要求的东西。
