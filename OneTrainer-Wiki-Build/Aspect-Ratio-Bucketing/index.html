    理解宽高比（AR）分桶：简单指南

    您拥有的：

    • 训练分辨率：1024 像素
    • 批量大小：4
    • 像素限制（又称预算）：1,048,576 像素
    • （因为 1024×1024=1,048,576）

    正确处理 AR = 更好的图像生成。OneTrainer 使用宽高比分桶。以下是它的工作原理。

    创建宽高比分桶

    • 使用 all_possible_input_aspects 相对于训练分辨率定义分桶

    读取每张图像

    • 程序查看数据集中的每张图像并记录宽度和高度

    为每张图像找到最佳分桶

    • 对于每张图像，程序找出哪个分桶是最接近的匹配

    调整图像以适应分桶

    • 缩放：如果图像仍然太大，它会缩小（按比例缩小）图像以适应像素预算。
    • 裁剪：
      • 如果仅缩放无法使其适应，那么它可以均匀裁剪一个维度（宽度或高度）。裁剪量在功能上 受分桶数量和训练分辨率的限制（我们从训练分辨率推导每个分桶的分辨率）。如果 裁剪抖动增强被启用，它将在一个或多个维度上随机分布所需的裁剪

    总之，我们尝试对图像进行尽可能小的调整。

    all_possible_input_aspects（宽度，高度）

    • (4.0, 1.0),
    • (3.5, 1.0),
    • (3.0, 1.0),
    • (2.5, 1.0),
    • (2.0, 1.0),
    • (1.75, 1.0), 常见宽屏（16/9）
    • (1.5, 1.0),
    • (1.25, 1.0),
    • (1.0, 1.0), 正方形
    • (1.0, 1.25),
    • (1.0, 1.5), 常见竖版
    • (1.0, 1.75),
    • (1.0, 2.0),
    • (1.0, 2.5),
    • (1.0, 3.0),
    • (1.0, 3.5),
    • (1.0, 4.0)

    示例"数据集"4 张图像

    • 16:9
      • 1280 x 720
      • 1366 x 768
      • 1600 x 900
      • 1920 x 1080

    让我们用 1920 x 1080 作为示例

    16:9 - 1920 x 1080（2.073M 像素）示例

    1. 确定 AR - 图像宽度除以高度 = 1.78
    2. 查看可用的分桶，我们最接近的匹配是 1.75:1，但它不是精确匹配
    3. 图像也超过了我们的像素预算，所以我们按比例缩小图像到 1365 × 768（1.048M 像素）
    4. 现在为了使其适应 1.75:1 分桶，我们必须减少其宽度，所以我们从宽度裁剪 21 像素

    结论

    • 宽高比分桶是调整到像素预算的宽高比。
    • 图像被缩放/裁剪以匹配最接近的可能分桶。
    • 在训练期间，一个批次只能用同一分桶上的图像填充。这解释了使用大于 1 的批量大小和不同比例的图像时可能的图像下降。
