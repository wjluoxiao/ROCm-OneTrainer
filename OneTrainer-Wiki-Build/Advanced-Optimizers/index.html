    Advanced Optimizers

    这些是配备了学术研究论文中最新、高级技术和方法的优化器。

    每个功能都应该独立工作，但同时使用多个功能是实验性的，因为它们可能以有益或有害的方式相互作用。

    🔥 最新动态

    （AdvOptm 2.2.2 版本）

    • Simplified AdEMAMix 现在对于所有 alpha grad 和 beta1 值都具有与 AdamW 相同的 LR！

    （AdvOptm 1.2.x 版本）

    • 添加了带有最新论文功能和设置的 Muon 优化器高级变体，请参阅：正交优化器 页面
    • 为所有 adv 优化器添加了谨慎权重衰减，请参阅：谨慎权重衰减 部分

    包含的优化器：

    • 在 UI 中带有 _ADV 后缀（Simplified_AdEMAMix 是例外）。
    优化器 描述 最适合
    Adam_Adv 高级 Adam 实现 通用
    Adopt_Adv 具有独立 beta2 的 Adam 变体 小批量大小制度的稳定训练
    Prodigy_Adv 带 D 适应的 Prodigy 带自动 LR 调优的 Adam
    Simplified_AdEMAMix 带累加器动量的 Adam 变体 正确调优时的小/大批量训练
    Lion_Adv 高级 Lion 实现 通用

    包含的功能：

    A. 通用安全功能

    这些功能适用于所有优化器，通常可以安全启用。

    功能 描述 推荐用法 性能影响 理论基础 兼容性
    融合反向传播 融合反向传播；梯度立即使用，内存即时释放。 内存受限环境 减少峰值内存 内存优化 所有优化器
    随机舍入 用随机舍入替换最近舍入，以在 BF16 中保留小梯度更新。 BF16 训练 最小开销（<5%） 重新审视 BFloat16 训练 所有优化器
    OrthoGrad 移除与权重平行的梯度分量以减少过拟合。 不带权重衰减的全微调 对于有效 BS=4，+33% 时间开销。EBS 越大 = 开销越小。 边缘处的 Grokking 所有优化器
    Factored 通过优化器状态的秩-1 1位分解实现内存高效优化。 大模型 / 内存受限硬件 增加压缩开销 SMMF 所有优化器
    谨慎权重衰减 仅当更新方向与参数符号对齐时才应用权重衰减的修改。 通用（需要 权重衰减 > 0） 无开销 谨慎权重衰减 所有优化器

    B. 单个功能：

    功能 描述 推荐用法 性能影响 理论基础 兼容性
    Cautious 仅当梯度方向与动量方向对齐时才应用更新。 加速收敛 无开销 C-Optim Adam/Adopt/Prodigy/Lion
    Grams 纯粹从当前梯度导出的更新方向。 当 Cautious 不足时 无开销 Grams Adam/Adopt/Prodigy
    AdEMAMix 双 EMA 系统，在数万步中保留梯度的相关性，加速收敛并减少遗忘 长训练运行，特别是在模型遗忘是一个问题的地方 +1 状态内存 AdEMAMix Adam/Adopt/Prodigy
    Simplified_AdEMAMix 基于累加器的动量，AdEMAMix 的单 EMA 变体 正确调优时的所有场景 无开销 连接 Adam/Adopt/Prodigy
    atan2 带有内置梯度裁剪的稳健 epsilon 替代 用于稳定的有界更新（或用于 Adopt，因为它需要这个） 无开销 Adam-atan2 Adam/Adopt/Prodigy
    Kourkoutas-β 基于梯度"太阳尖峰"比率的逐层自适应 β₂ 嘈杂/小 BS/高 LR 训练 最小开销 Kourkoutas-β Adam/Adopt/Prodigy/Simplified_AdEMAMix

    功能详情

    • 本教程针对小批量大小制度（1-64），因为这是 OT 的最大受众。
    • 大批量大小训练（≥512）应进行不同的调优。

    Cautious 和 Grams：

    这两个共享相同的核心思想，即相对于原始梯度修改动量：

    • Cautious 通过掩蔽和等待动量在更新前将其方向与原始梯度对齐来温和地做到这一点。

    • Grams 更激进，因为它迫使动量采用梯度的符号。

    Grams 在其论文中被认为更好，但如果您愿意，可以使用其中一个。 * 如果您同时启用两者，Grams 将被启用，Cautious 将被禁用。

    AdEMAMix：

    此功能为优化器添加了第二个 EMA，它衰减非常缓慢，并在数万步中保留记忆（取决于 beta3 值）。

    AdEMAMix 值得更多关注，原因如下：

    1. 学术研究论文的理论和经验都证明，Adam 的第一矩对于小批量大小几乎没有用。您可以只使用第二矩，并且看不到任何区别。

    参考：请查看论文中的"动量和批量大小"部分：
    AdaMeM: Adafactor 的内存高效动量
    - 因为它涵盖了大多数陈述这一点的论文。

    1. 另一方面，AdEMAMix 的 EMA 已被证明对小批量大小非常有益。在这些场景中，您可以完全跳过 Adam 的 EMA，只使用 AdEMAMix 的 EMA。
    AdEMAMix 的可调超参数
    参数 默认值 调优指南
    beta3 0.9999 应根据训练长度选择：
    • 运行 >120k 步：保持在 0.9999
    • 运行 ≤120k 步：尝试 0.999
    注意：我默认将其保持在 0.9999，但本指南遵循 AdEMAMix 论文的建议。
    alpha 5 AdEMAMix 的 EMA 贡献的乘数。
    • 值 5 可能很高 - 考虑降低 LR。
    • 如果训练发散，减少到 2-3。
    • 增加以加强 AdEMAMix 的效果；减少以减弱它。
    > ⚠️ 注意 AdEMAMix 需要比 Adam 低 2-4 倍的学习率才能保持稳定，因为它的更新步骤更大。

    Simplified_AdEMAMix：

    这是 Simplified_AdEMAMix 论文（arXiv:2502.02431） 中引入的一种有趣且独特的方法。 既然 Adam 的 EMA 已被证明有益于大批量大小，而 AdEMAMix 的慢 EMA 有益于小批量大小，为什么不对齐这两种方法呢？ 这正是论文提出的：它用一个梯度累加器替换了 Adam 的第一矩（EMA），捕捉了两种方法的优势。论文从理论和经验上证明 - 当正确调优时，这个单一的修改累加器可以匹配甚至超过 Adam 和 AdEMAMix 的效率。

    📌 论文的关键见解：
    经典动量方法（如 Adam 的 EMA）在嘈杂制度（小 BS）中不会加速优化器。
    Simplified_AdEMAMix 中的累加器 + Grad α 设计直接解决了这个问题：
    - 累加器通过逐步累积梯度来加速优化器。
    - Grad α 缩放原始梯度，允许对最近的更新进行可控强调，非常适合在小批量训练中穿透噪音。

    ⚠️ 重要兼容性说明：
    由于其对第一矩的根本性修改： - 它与依赖标准动量的功能（如 Grams、Cautious）不兼容。

    Simplified_AdEMAMix 的可调超参数
    参数 默认值 调优指南
    beta1 0.99 现在控制累加器的有效内存长度（以前是 EMA 衰减）。
    • 0.9 → ~10 步内存 → 对于小批量太短。
    • 对于小批量大小，使用 0.99 到 0.9999，取决于训练长度和所需的稳定性。
    Grad α 平滑因子 100 最关键的调优参数。 控制最终更新中原始、当前梯度的权重。
    • 应与批量大小成反比缩放（即随着批量大小增加线性减少）。
    • 高值（10-100）：强调最近的梯度，非常适合小批量穿透噪音并快速适应。
    如何调优 Simplified AdEMAMix

    这是 Sim AdEMAMix 超参数的简单指南。使用这些规则作为近似值来找到您的初始起始值。

    beta1（长期动量）

    目标是设置一个相对于您的数据集大小具有半衰期的长期动量。 * 计算有效步骤： effective_steps = total_images / (batch_size * accumulation_steps)

    • 计算 beta1： beta1 = 1 - (1 / (effective_steps / 2))

    这确保 beta1 每次更新覆盖您数据集的一半。

    Alpha Grad

    alpha grad 可以通过假设它与您的总批量大小线性缩放来简化。我们使用 alpha = 100 作为批量大小为 1 的基线。 * 公式： alpha_grad = 100 / (batch_size * accumulation_steps)

    实际示例 如果您有 880 张图像，批量大小为 2，累积步骤为 4： * 有效步骤：880 / (2 * 4) = 110 * beta1：1 - (1 / 55) = 0.98 * Alpha Grad：100 / (2 * 4) = 12.5

    这些值提供了坚实的基线。

    Atan2：

    一种简单而有效的 Adam 基优化器方法，它改进了 eps 超参数，完全替换了它并使优化器具有尺度不变性。

    它将更新步骤限制在 [-2, 2] 范围内，有效地充当内置的更新裁剪，以防止优化过程中的大型、不稳定的跳跃。

    ✅ 强烈推荐用于不稳定的优化器：
    这对于像 Adopt_Adv 这样的优化器特别有益，它们通常需要高 eps 值和更新裁剪才能在训练期间保持稳定。

    Kourkoutas-β

    Kourkoutas-β 引入了一种太阳尖峰驱动的、逐层自适应的第二矩衰减（β₂），作为 Adam_Adv、Adopt_Adv、Prodigy_Adv 和 Simplified_AdEMAMix 的可选增强。

    它不使用固定的 β₂（如 0.999 或 0.95），而是基于有界的太阳尖峰比率 逐层动态调制 β₂：

    • 在梯度爆发期间 → β₂ ↓ 向 较低 β₂ → 更快反应
    • 在平静阶段 → β₂ ↑ 向 选定的 β₂ → 更强的平滑

    这对于嘈杂训练、小批量大小和高学习率特别有效，其中梯度范数由于噪音或激进的 LR 调度而突然变化。

    优点/缺点
    类别 详情
    ✅ 优点 • 逐层适应融合了高 β₂（强平滑）和低 β₂（快速反应）的好处。
    • 对突然的损失景观变化具有鲁棒性，在梯度爆发期间快速反应，在平静阶段平滑。
    • 对激进的学习率具有高容忍度。
    ⚠️ 缺点 • 在训练开始时可能不稳定，因为早期梯度范数不可靠；通过使用 K-β 预热步骤来缓解。

    💡 最佳实践：将 K_warmup_steps 设置为等于您的标准 LR 预热步骤。在预热期间，优化器使用静态的 beta2；仅在预热结束后才开始适应。

    用法
    • 启用 Kourkoutas Beta
    • 设置 beta2 = 0.999 并让它适应。

    谨慎权重衰减（CWD）：

    这是 一项新技术，它改进了权重衰减的应用方式。

    在标准优化（如 AdamW 或 Lion）中，权重衰减总是将参数拉向零，不管损失函数想要什么。这经常造成冲突，即优化器想要增加一个值，但权重衰减通过试图减少它来对抗它。

    CWD 通过应用一个简单的逻辑掩码来解决这个问题： * 如果优化器更新和参数符号对齐（一致）： 应用权重衰减以帮助缩小值。 * 如果它们不一致（冲突）： 关闭该特定参数的权重衰减，让优化器完成它的工作。

    与"Cautious 变体"（C-Optim）的比较

    不要将 Cautious 与 谨慎权重衰减混淆： * Cautious（C-Optim）： 基于动量对齐掩蔽优化器更新。 * 谨慎权重衰减（CWD）： 基于参数对齐掩蔽权重衰减。 * 您可以同时使用两者吗？ 是的。 它们作用于步函数的不同部分。

    用法
    • 只需启用它并将您的 权重衰减 设置为 > 0 的值。
    • 无需调优： 它使用与您通常使用的完全相同的 权重衰减 值。
    • 性能： 论文证明在视觉和 LLM 任务上都始终具有更低的最终损失和更好的泛化能力。

    Factored 注意事项：

    • 当 Factored 启用（True）时，它将分解所有优化器状态。无论它们属于 AdEMAMix 的慢 EMA、Simplified_AdEMAMix 的累加器还是 Lion 的动量。

    • 优化器对 Factored 1 位分解的容忍度排名：

    • Lion：最容忍 - 在我的测试中产生相同的结果。
    • Simplified_AdEMAMix：下一个 - 由于其基于原始、未压缩梯度的大更新，它应该产生相同或非常可比的结果。
    • Adam / Prodigy / Adopt：提供可比的结果 - 并且仍然值得用于内存节省（优化器状态小 16-32 倍）。

    Prodigy_Adv 注意事项：

    • PRODIGY_Adv 超参数注意事项：
    • beta3 = Prodigy 计算的 beta3；留空，不会有伤害。
    • beta3 EMA = AdEMAMix 的 EMA 的 beta3，您可以根据运行长度进行调优。它仅在 AdEMAMix EMA 为 true 时生效。

    Lion_Prodigy_Adv 注意事项：

    • Lion 是一种基于符号的优化器，每步产生大更新。这也可能 destabilize Prodigy 计算，导致 LR 被高估。在较新版本中，我们为其添加了选项 d_limiter。这默认是 True，应该保持 True，因为它迫使 Prodigy 与 Lion 良好配合，产生合适的 LR。

    其他注意事项：

    • 对于优化器（Adam_Adv、ADOPT_Adv、Prodigy_Adv）：当 beta1 设置为 0（或未设置/None）时，优化器将完全跳过第一矩（标准 EMA）。
      ✅ 这与 use_AdEMAMix 选项完全兼容，允许您只使用 AdEMAMix 的慢 EMA，同时绕过 Adam 的传统动量。非常适合标准 EMA 几乎没有价值的小批量训练

    • Lion 是与 AdEMAMix/Simplified_AdEMAMix 关系最密切的优化器，因为其典型设置为 beta1 < beta2（如 0.9 和 0.99）。与历史动量相比，这给原始梯度提供了大约 10 倍的权重。行为类似于 alpha=10 的 Simplified_AdEMAMix，但使用符号操作而不是自适应学习率缩放。

    • 跨优化器的更新规则差异：

    • AdEMAMix：
      update = fast_EMA + (α * slow_EMA)
      → 将短期 EMA 与放大的长期 EMA 混合。
    • Simplified_AdEMAMix：
      update = (α * grad) + accumulator
      → 直接缩放当前梯度并将其添加到衰减的累加器中。

    • Adam：
      update = first_moment_EMA
      → 梯度的标准指数移动平均。

    • 可跳过超参数的注意事项：

    • 当 AdEMAMix EMA 为 False 时，beta3 EMA 和 alpha 参数将被跳过并且没有效果。
    • 当 Simplified_AdEMAMix 为 False 时，grad α 参数将被跳过并且没有效果。

    ⚙️ 配置指南

    🤏 "我的批量大小很小"配置：

    • 优化器： ADOPT_Adv（最初为小 BS 提出）。
    • Simplified AdEMAMix： True
    • a Grad： 100.0（高响应性，对小 BS 有益）。
    • beta1： 0.99 到 0.9999（长期动量，对小 BS 有益）。
    • Atan2 缩放： True（为 ADOPT 保持此设置开启，在 ADOPT 情况下它适用于 Simplified AdEMAMix）。
    • Kourkoutas Beta： True（为嘈杂/不稳定训练或/和不良调优提供更多稳定性）。
    • 学习率： 与往常相同。
    • 权重衰减： 与往常相同

    🥔 "我 VRAM 低"配置：

    • 优化器： 任何 _Adv 优化器
    • Factored 优化器： True
    • 融合反向传播： True
    • 为了进一步减少 VRAM，您可以在基于 Adam 的优化器中将 beta1 设置为 0，但是这将完全跳过动量。

    🧙‍♂️ "小 BS 的旧魔法预设"配置：

    这是（以前）对我来说效果最好的预设，并在我的（旧）测试（微调/LoRA/嵌入）中提供了最强/最快的结果：

    学习率: 1
optimizer: PRODIGY_Adv
settings:
  - beta1: 0.99 # 控制动量衰减，~100 步有效内存。根据训练长度和稳定性需求调整为 0.999（1000 步）或 0.9999（10000 步）。
  - beta2: 0.999 # 因为我们将使用 k-b。
  - Simplified_AdEMAMix: True
  - Grad α: 100 # 控制更新中原始梯度的权重
  - OrthoGrad: True
  - weight_decay: 0.0
  - d_coef: 1
  - Kourkoutas Beta: True
  - K-β 预热步骤: 20 # 在 20-200 范围内或您通常用于 LR 的预热步骤数。
  - factored: False # 可以是 true 或 false，由于 Simplified_AdEMAMix 对 1 位分解的高容忍度，质量不应下降。

    ✅ 为什么这效果好： - beta1=0.99 稳定了动量，同时允许 Simplified_AdEMAMix 通过原始梯度主导更新方向。 - beta2=0.999 和 Kourkoutas Beta 稳定了优化器，同时逐层适应 beta2。 - Grad α=100 强调最近的梯度 - 非常适合小批量、嘈杂制度。 - OrthoGrad 防止过拟合而不需要权重衰减（设置 weight_decay=0.0）。或者您可以为 LoRA 和嵌入使用 weight_decay，因为对它们是安全的。 - factored=True 启用优化器状态的 1 位分解，提供 16-32 倍内存减少，精度损失最小。这要归功于 Simplified_AdEMAMix 对状态压缩的固有容忍度（因为其更新严重依赖原始、未压缩的梯度）。
